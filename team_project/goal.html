THE HIVE

Project Goal

The goal of the group project was to practice various aspects of web development and design while working with a team.

Project Description

A new modern honey company that provides customers with a unique and informative shopping experience that goes beyond purchasing honey.

Project Purpose

This website allows customers to view product information, contact the company directly, and make purchases in the future.

Team Goals & Outcomes

Implement a responsive layout for all screen sizes - MET

We made sure our layout adapted to desktop, tablet, and mobile using CSS Flexbox and media queries.

Use JavaScript for dynamic functionality (e.g., loading navigation/footer) - MET

We successfully used JavaScript to dynamically load the nav and footer to avoid code duplication.

Ensure semantic HTML structure across all pages — MET (mostly)

We used semantic tags like <header>, <main>, and <footer>, though we had a few structure issues at first that were corrected later.

Write clean, well-organized CSS for maintainable styling — MET

We kept our CSS organized by page and used consistent naming conventions for readability.

Have at least one “extra” feature (e.g., animations, product gallery, or form validation) — MET

We added a working contact form with validation and an embedded narrated PowerPoint, which went beyond basic requirements.

Update the site over time to include more dynamic JavaScript features — MET

We improved our project throughout the quarter by moving repeated elements (nav/footer) to JavaScript templates and adding client-side form validation. These steps made the site cleaner and more maintainable.

Collaborate effectively as a team using GitHub — SEMI MET

We used GitHub for hosting and updates, but didn’t fully utilize branches or pull requests. Some changes were committed directly to main, which sometimes caused merge conflicts.
